# 02 — Chunking: нарезка документов

**Время: ~2 часа** · Файл: `rag/chunker.py`

## Цель модуля

На выходе — `chunk_documents()`, превращающая `list[Document]` в `list[Chunk]` с метаданными (из какого файла, из-под какого заголовка). Ты на своих данных прочувствуешь главный trade-off RAG: маленькие чанки точнее находятся, но теряют контекст; большие сохраняют контекст, но «размывают» вектор и хуже матчатся с коротким вопросом.

## Теория-минимум

Ключевое ограничение — **модель эмбеддингов**: `all-MiniLM-L6-v2` обрезает вход после 256 токенов (~190 слов) **молча**. Всё, что дальше, просто не попадёт в вектор. Поэтому наш потолок — примерно 200 слов на чанк.

Стратегии: фиксированное окно (просто, но режет мысли посередине), окно с overlap (перекрытие спасает разрезанные предложения), структурная нарезка (по заголовкам — куски семантически цельные). Мы совместим: сначала режем документ на секции по заголовкам, потом длинные секции — скользящим окном.

Углубиться: [Pinecone: Chunking Strategies](https://www.pinecone.io/learn/chunking-strategies/).

## Что реализовать

В `rag/config.py` добавь: `CHUNK_SIZE = 180` (слов), `CHUNK_OVERLAP = 40` (слов).

```python
@dataclass
class Chunk:
    chunk_id: int    # сквозной номер по всей коллекции
    doc_path: str    # "tutorial/query-params.md"
    heading: str     # ближайший заголовок секции (или title документа)
    text: str
```

1. `def split_into_sections(doc: Document) -> list[tuple[str, str]]` — режет текст по заголовкам уровня `## ` на пары `(heading, section_text)`. Текст до первого `##` — секция с heading = title документа. **Не разрезай внутри блока кода**: строка ```` ``` ```` переключает флаг «внутри кода», и `##` внутри кода — не заголовок.
2. `def chunk_text(text: str, chunk_size: int, overlap: int) -> list[str]` — скользящее окно по **словам** (`text.split()`): окно `chunk_size`, шаг `chunk_size - overlap`. В начале — `assert 0 <= overlap < chunk_size`.
3. `def chunk_documents(docs: list[Document], chunk_size: int, overlap: int) -> list[Chunk]` — для каждого документа: секции → каждую секцию через `chunk_text` → собрать `Chunk`'и со сквозным `chunk_id`. В текст чанка **допиши префикс** `f"{doc.title} / {heading}\n\n"` — дешёвый трюк, заметно улучшающий retrieval: контекст заголовка попадает в вектор.

**Edge cases:** секция короче `chunk_size` (один чанк целиком, без окна); пустая секция (пропустить); хвост окна из 2–3 слов (если последний кусок короче `overlap` — приклей к предыдущему или выкинь); документ вовсе без `##`.

## Пошаговая инструкция

1. Сначала `chunk_text` — чистая функция, отладь её на игрушечной строке из 10 слов с `chunk_size=4, overlap=1`, посчитай ожидаемые окна **на бумаге** и сравни.
2. Потом `split_into_sections` с флагом кода. Проверь на документе, где точно есть код с `#`-комментариями (почти любой tutorial).
3. `chunk_documents` — сборка. Выведи статистику: всего чанков, мин/медиана/макс длина в словах, топ-5 самых длинных.
4. **Грабли:** слова ≠ токены. 180 слов ≈ 230–250 токенов — уже на грани. Код в чанках токенизируется «дороже» слов, так что часть хвостов всё равно обрежется — для учебного проекта приемлемо, но знай об этом (вопрос на собеседовании).
5. **Грабли:** бесконечный цикл в окне при `overlap >= chunk_size` — от этого и стоит assert.

## Как проверить

```python
if __name__ == "__main__":
    # юнит: 10 слов, окно 4, overlap 1 -> старты 0, 3, 6 -> 3 чанка
    words = [f"w{i}" for i in range(10)]
    pieces = chunk_text(" ".join(words), chunk_size=4, overlap=1)
    assert len(pieces) == 3, pieces
    assert pieces[0].split() == ["w0", "w1", "w2", "w3"]

    docs = load_documents(DOCS_ROOT)
    chunks = chunk_documents(docs, CHUNK_SIZE, CHUNK_OVERLAP)
    assert len(chunks) > 1000
    assert all(len(c.text.split()) <= CHUNK_SIZE + 30 for c in chunks)  # +30 на префикс заголовка
    assert len({c.chunk_id for c in chunks}) == len(chunks)  # id уникальны
```

Глазами: распечатай 3 случайных чанка — каждый должен быть осмысленным куском, с понятным заголовком в префиксе.
