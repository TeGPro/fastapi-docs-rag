# 05 — Промпт и генерация ответа (Groq)

**Время: ~2.5 часа** · Файл: `rag/generator.py`

## Цель модуля

На выходе — функция `answer_question()`: вопрос → retrieval → промпт с контекстом → Groq LLM → ответ с указанием источников. Ты научишься главному навыку RAG-промптинга — **grounding**: заставить модель отвечать только по контексту, ссылаться на источники и честно отказываться, когда ответа в контексте нет.

## Теория-минимум

LLM получает список сообщений: `system` (постоянные правила поведения) и `user` (найденные чанки + вопрос). Всё «поведение RAG» — запрет на выдумывание, формат цитат, отказ — задаётся текстом system-промпта. Второй рычаг — `temperature=0`: детерминированнее ответы, проще отлаживать и оценивать.

Groq — это API-провайдер с очень быстрым инференсом открытых моделей и бесплатным тиром. Берём `llama-3.3-70b-versatile`. SDK `groq` совместим по интерфейсу с OpenAI: `client.chat.completions.create(model=..., messages=[...])`. У бесплатного тира есть **rate limit** (запросов/токенов в минуту) — для разработки и eval хватает, но в цикле оценки, возможно, понадобится пауза между запросами.

Углубиться: [Groq quickstart](https://console.groq.com/docs/quickstart), [rate limits](https://console.groq.com/docs/rate-limits).

## Пошаговая инструкция

1. Сначала «hello world» с Groq без RAG: 5 строк, один вопрос — убедись, что ключ и SDK работают.
2. Потом `format_context` и `build_messages` — чистые функции. **Распечатай собранный промпт целиком** и прочитай глазами: это лучший дебаг-инструмент модуля.
3. `generate`, затем `answer_question`.
4. Прогони 5 смоук-вопросов из модуля 04 — теперь с генерацией. Читай ответы критически: совпадает ли с тем, что ты знаешь о FastAPI?
5. **Грабли:** свалить все чанки в кучу без разделителей и номеров — модель не сможет цитировать. Нумерация в `format_context` — это то, что делает `[1]` в ответе возможным.
6. **Грабли:** слабый system-промпт → на вопрос вне доков модель радостно ответит из своих знаний — это уже не RAG. Тестовый вопрос-ловушка обязателен (см. проверку).
7. **Грабли:** не хардкодь ключ в коде — только `GROQ_API_KEY` из окружения.

## Как проверить

Три ручных теста (в `__main__`):

1. **Обычный вопрос:** "How do I declare optional query parameters?" → связный ответ, соответствует докам, есть `[n]`-цитаты.
2. **Вопрос-ловушка (не из доков):** "What is the capital of France?" → модель отказывается отвечать, а не говорит «Paris».
3. **Вопрос про смежное, но отсутствующее:** "How do I use FastAPI with Django ORM?" → отказ или честное «в контексте нет», без фантазий.

```python
ans = answer_question("What is the capital of France?", embedder, store)
assert "paris" not in ans.text.lower(), "модель ответила из своих знаний — усиль system-промпт"
```
