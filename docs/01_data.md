# 01 — Загрузка и парсинг документов

**Время: ~1.5 часа** · Файл: `rag/data_loader.py` (+ константы в `rag/config.py`)

## Цель модуля

На выходе — функция `load_documents()`, которая превращает склонированный репозиторий FastAPI в список чистых объектов `Document` (путь, заголовок, текст). Ты поймёшь, что «загрузка данных» в RAG — это на 90% фильтрация мусора: правильно выбрать, *что не брать*, важнее, чем парсинг сам по себе.

## Теория-минимум

Документация FastAPI лежит в `data/fastapi_repo/docs/<язык>/docs/**/*.md`. Берём **только `en`** — наша модель эмбеддингов англоязычная. Файлы — обычный Markdown, но с особенностями: директивы включения примеров кода вида `{* ../../docs_src/... *}` (код подставляется при сборке сайта — в сыром файле его нет), HTML-вставки (`<img>`, `<div>`), и несколько служебных файлов-гигантов (`release-notes.md` — многотысячестрочный changelog, который замусорит любой поиск).

Углубиться: [pathlib — официальные доки](https://docs.python.org/3/library/pathlib.html) (`rglob` — твой главный инструмент).

## Что реализовать

В `rag/config.py`:

```python
DOCS_ROOT = Path("data/fastapi_repo/docs/en/docs")
EXCLUDE_FILES = {"release-notes.md", "newsletter.md", "management-tasks.md"}
```

В `rag/data_loader.py`:

```python
@dataclass
class Document:
    path: str    # путь относительно DOCS_ROOT, напр. "tutorial/query-params.md"
    title: str   # текст первого заголовка "# ..." или имя файла, если H1 нет
    text: str    # очищенный текст
```

1. `def find_markdown_files(docs_root: Path) -> list[Path]` — все `*.md` рекурсивно, минус `EXCLUDE_FILES`. Отсортируй — стабильный порядок упростит отладку.
2. `def clean_markdown(raw: str) -> str` — убрать строки-директивы `{* ... *}` и `{!...!}`, убрать HTML-теги (одной регуляркой `<[^>]+>`), схлопнуть тройные+ пустые строки в двойные. **Блоки кода (```) оставляем** — в них половина пользы документации.
3. `def parse_document(path: Path, docs_root: Path) -> Document | None` — прочитать файл (`encoding="utf-8"`), очистить, вытащить title. Вернуть `None`, если после очистки текст пустой или короче ~200 символов (обложки-заглушки не нужны).
4. `def load_documents(docs_root: Path) -> list[Document]` — обойти файлы, собрать не-None документы.

**Edge cases:** файл без H1 (title = имя файла без расширения); пустой файл; H1 не в первой строке (ищи первую строку, начинающуюся с `"# "`, а не только `lines[0]`).

## Пошаговая инструкция

1. Начни с `find_markdown_files` — простая, сразу проверяемая: выведи `len()` и первые 5 путей.
2. Потом `clean_markdown` — держи её чистой функцией `str -> str`, так её тривиально тестировать без файлов.
3. `parse_document`, затем `load_documents` — тонкий цикл поверх готовых частей.
4. **Грабли:** не пытайся парсить Markdown «по-настоящему» (AST, markdown-it). Для RAG текст с остатками разметки — нормально: `##` и `-` не мешают ни эмбеддингам, ни LLM. Достаточно убрать явный мусор.
5. **Грабли:** забыть исключить `release-notes.md` → потом в retrieval на любой вопрос будут вылезать куски changelog'а. Если увидишь такое в модуле 04 — вернись сюда.

## Как проверить

Внизу файла — блок самопроверки:

```python
if __name__ == "__main__":
    docs = load_documents(DOCS_ROOT)
    assert len(docs) > 100, f"слишком мало документов: {len(docs)}"
    assert all(d.text.strip() for d in docs)
    assert all(not d.path.startswith("/") for d in docs)  # пути относительные
    assert "{*" not in docs[0].text
```

Плюс глазами: `print` title и первые 300 символов у 3 случайных документов — текст должен читаться как связная документация, без HTML и директив.

## Критерий готовности

- [ ] `python -m rag.data_loader` отрабатывает без ошибок, все assert'ы проходят.
- [ ] 150±50 документов; `release-notes.md` отсутствует.
- [ ] У каждого документа осмысленный `title` и относительный `path`.
- [ ] Случайный документ при просмотре глазами — чистый читаемый текст.
